规则2（Study仓库目录与归档规范）

适用范围
- 仅在当前工作目录以 `/Users/atmoos/Documents/DATA/Obsidian/Study` 开头时生效。
- 本规则用于定义目录职责与归档位置；执行优先级遵循规则1。

仓库定位
- Study 仓库用于学习知识、记录生活、复盘与迭代沉淀。
- 处理任务时需区分仓库用途与权限边界，避免误写到非目标仓库。

跨仓库操作
- 若任务涉及 Study 之外的仓库或路径，先明确提示“跨仓库操作”，并在得到用户确认后再执行。
- 未确认前，仅提供建议路径或操作方案，不直接写入外部仓库。

目录职责
- 以下目录职责用于当前阶段参考；后续如仓库文件夹发生变更，以仓库最新实际结构为准。
- `0.Basic-Life`：日记、周记、模板、生活复盘与阶段性总结。
- `1.PDF-Note`：PDF 阅读与批注记录（与 `Pdf` 目录联动）。
- `2.AI`：AI 相关沉淀（提示词、工作流、实践经验、技术与感想）。
- `3.Code`：编程学习笔记、开发技巧、技术沉淀。
- `4.Study-Exam`：考研 11408 相关学习内容与讨论结论。
- `5.Web`：收录从各处整理来的 Markdown 资料；暂时无法明确归类的内容可先放此处，后续再二次整理。
- `Pdf`：PDF/电子书等原始资料库。

周/月总结归档
- 周总结：写入当周周记的“AI汇总”标题下。
- 月总结：归档到 `0.Basic-Life/3.DayNote/月记` 目录。
- 月记命名规则：`YYYY-M月.md`（示例：`2026-4月.md`）。
- 月记模板：`0.Basic-Life/1.模版/月记模板.md`。

触发与更新规则
- 当用户明确提出“周总结/本周复盘”时执行周总结。
- 当用户明确提出“月总结/本月复盘”时执行月总结。
- 若目标月记文件已存在，优先更新原文件，不重复创建。
- 若目标目录不存在，先询问是否创建；未确认前不创建目录。
